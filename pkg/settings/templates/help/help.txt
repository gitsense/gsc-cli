GSC-WORKSPACE(1)             GitSense Manual            GSC-WORKSPACE(1)

NAME
    gsc-workspace - Interactive AI-generated code environment

DESCRIPTION
    This is a temporary "shadow" directory containing code generated by
    the AI. You can run, modify, and test this code safely before
    applying it to your main project. This environment is managed by the
    'gsc ws' command.

DIRECTORY STRUCTURE
    The workspace is organized into two main categories:

    mapped/
        Contains files that were successfully mapped to your project
        structure. The directory hierarchy mirrors your project root.
        Example: mapped/src/utils/helper.go

    unmapped/
        Contains files that could not be mapped to a specific project
        location (e.g., new files or snippets). These are organized
        using a position-based naming convention to ensure shell safety
        and natural sorting.

        components/
            Organized by position and sanitized name.
            Single Message Mode: 01_sanitized_name/
            Full Contract Mode: 1_01_sanitized_name/ (msg_position_name)
            Example: unmapped/components/01_hello_world/

        snippets/
            Small code blocks without a component name.
            Example: unmapped/snippets/generated_001.go

        patches/
            Raw patch files for reference.
            Example: unmapped/components/01_hello_world_patch_a1b2c3d4.diff

    File Naming Convention:
        - mapped/ and components/ directories contain 'generated.<ext>' files.
        - snippets/ directories contain 'generated_<index>.<ext>' files.
        - 'source.<ext>' files represent the original project code (for mapped files).

    Sanitization Rules:
        - Spaces are replaced with underscores.
        - Special characters are removed (keeps a-z, 0-9, -, _).
        - Names are converted to lowercase.

COMMANDS
    .block [INDEX|list|next|prev|root]
        Navigate between code blocks in the current message. This is the primary
        tool for reviewing AI-generated code within the active workspace.

        Behavior (Smart Default):
        - If the workspace contains only one block, '.block' jumps directly to it.
        - If the workspace contains multiple blocks, '.block' launches an
          interactive picker (fzf) to select a block.

        Arguments:
        INDEX       Jump directly to the block at the specified 1-based index.
        list        Force the interactive picker (fzf).
        next        Jump to the next block relative to the current directory.
        prev        Jump to the previous block relative to the current directory.
        root        Return to the workspace root directory.

        Examples:
        .block              # Smart jump or picker
        .block 2            # Jump to the second block
        .block next         # Move to the next block
        .block root         # Return to workspace root

    .map [--all]
        Visualizes the workspace hierarchy. By default, it provides a focused
        view of the current workspace, collapsing others into summaries.

        Options:
        --all       Expand all workspaces in the contract to show the full tree.

        Examples:
        .map                # Show focused view of current workspace
        .map --all          # Show full tree of all workspaces

    .goto
        Opens an interactive picker (fzf) to jump to any block across all
        messages in the contract. This allows for rapid navigation between
        different workspaces.

        Examples:
        .goto              # Open picker to jump to any block

    .ffp [OPTIONS]
        Fuzzy find files in the real project root ($GSC_PROJECT_ROOT).
        This is the primary tool for locating source files to compare
        against the AI-generated code in this workspace.

        Options:
        --diff              Diff the selected file against 'generated.*' in CWD.
        --from <file>       Specify the source file for the diff.
        --open              Open the selected file in your preferred editor.
        --copy              Copy the content of the selected file to the clipboard.
        --relative          Copy the relative path (from project root) to the clipboard.
        --path              Copy the absolute path to the clipboard (default).

    .send [COMMENT]
        Reads from standard input and sends the content back to the
        active GitSense Chat session.

        Options:
        --visibility [human-only | human-public]
            Set who can see the message. Default is human-public.

        --wrap [LANGUAGE]
            Wraps the captured output in a Markdown code block using
            the specified language (e.g., c, go, bash, text).

        --md-before [TEXT]
            Prepends the specified Markdown text before the output.

        --md-after [TEXT]
            Appends the specified Markdown text after the output.

    .help
        Displays this detailed manual.

ENVIRONMENT
    GSC_CHAT_ID              The ID of the chat receiving 'send' output.
    GSC_CONTRACT_UUID        The UUID of the active contract.
    GSC_CONTRACT_MAPPED_ROOT The root directory containing all shadow workspaces for the contract.
    GSC_PROJECT_ROOT         The absolute path to the actual project tied to the contract.

EXIT
    Type 'exit' to close this session and return to your project root.
