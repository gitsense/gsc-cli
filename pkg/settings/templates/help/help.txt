GSC-WORKSPACE(1)             GitSense Manual            GSC-WORKSPACE(1)

NAME
    gsc-workspace - Interactive AI-generated code environment

DESCRIPTION
    This is a temporary "shadow" directory containing code generated by
    the AI. You can run, modify, and test this code safely before
    applying it to your main project.

DIRECTORY STRUCTURE
    The workspace is organized into two main categories:

    mapped/
        Contains files that were successfully mapped to your project
        structure. The directory hierarchy mirrors your project root.
        Example: mapped/src/utils/helper.go

    unmapped/
        Contains files that could not be mapped to a specific project
        location (e.g., new files or snippets). These are organized
        using a position-based naming convention to ensure shell safety
        and natural sorting.

        components/
            Organized by position and sanitized name.
            Single Message Mode: 01_sanitized_name/
            Full Contract Mode: 1_01_sanitized_name/ (msg_position_name)
            Example: unmapped/components/01_hello_world/

        snippets/
            Small code blocks without a component name.
            Example: unmapped/snippets/block_001.go

        patches/
            Raw patch files for reference.
            Example: unmapped/components/01_hello_world_patch_a1b2c3d4.diff

    Sanitization Rules:
        - Spaces are replaced with underscores.
        - Special characters are removed (keeps a-z, 0-9, -, _).
        - Names are converted to lowercase.

COMMANDS
    send [COMMENT]
        Reads from standard input and sends the content back to the
        active GitSense Chat session.

        Options:
        --visibility [human-only | human-public]
            Set who can see the message. Default is human-public.

        --wrap [LANGUAGE]
            Wraps the captured output in a Markdown code block using
            the specified language (e.g., c, go, bash, text).

        --md-before [TEXT]
            Prepends the specified Markdown text before the output.

        --md-after [TEXT]
            Appends the specified Markdown text after the output.

    save
        Copies the AI-generated files from this workspace into your
        actual project directory. Before overwriting, a backup of your
        original file is created in .gitsense/backups/.

    undo
        Reverts the most recent 'save' command by restoring the
        original file from the backup directory.

    diff [FILE]
        Launches your preferred diff tool (e.g., vimdiff) to compare
        the AI-generated version in this workspace with the original
        version in your project.

    help
        Displays this detailed manual.

ENVIRONMENT
    GSC_CHAT_ID         The ID of the chat receiving 'send' output.
    GSC_CONTRACT_UUID   The UUID of the active contract.
    GSC_PROJECT_ROOT    The absolute path to the actual project tied to the contract.
    GSC_MAPPED_WS_HASH  The unique hash of this workspace version.
    GSC_MAPPED_WS_ROOT  The absolute path to the root of this shadow workspace.

EXIT
    Type 'exit' to close this session and return to your project root.
